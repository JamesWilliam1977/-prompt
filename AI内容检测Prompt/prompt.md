# AI生成文本识别特征库(Author: 两斤)

## 语言层面特征

### 1. 句式模式 - "纠偏癖"
**核心表现：过度使用对比与修正句式**
- `"A不是X，而是Y"` / `"它不代表...它只代表..."`
- `"真正的X，不在于...而在于..."` / `"关键不是...而是..."`
- `"与其说...不如说..."` / `"问题不在于...而在于..."`

**深层原因：** AI具有"逻辑洁癖"和"定义冲动"，试图通过排除法确保表达精确性。人类更倾向于直接表达，而非先否定再肯定。

### 2. 标点使用 - "完美主义洁癖"
**引号滥用症：**
- 给普通概念强制加引号：`"护城河"`、`"性价比"`、`"内卷"`
- 对专业术语过度标记：`"范式"`、`"赋能"`、`"生态"`
- 喜欢给动词加引号强调：`"赋能"`、`"颠覆"`、`"重塑"`
- 连形容词都要引号化：`"核心"`、`"关键"`、`"重要"`

**破折号依赖症：**
- 过度使用破折号做解释：`核心问题——如何平衡效率与公平`
- 频繁用破折号制造停顿感：`这就是关键——理解用户真正的需求`
- 用破折号代替冒号、分号等其他标点
- 一段话里多次使用破折号，显得特别"文学腔"

**其他标点癖好：**
- **冒号强迫症**：`关键在于：`, `问题是：`, `答案很简单：`
- **分号过度使用**：在中文语境下不合时宜地频繁使用分号
- **感叹号谨慎症**：很少用感叹号，即使表达强烈情感也用句号
- **问号教学化**：喜欢用反问句引导：`那么问题来了：`、`这意味着什么？`
- **括号解释癖**：频繁用括号补充说明，生怕读者理解不了

**机械化完美：**
- 每句必有标准结尾标点，绝不省略
- 并列词汇严格使用顿号分隔，从不混用逗号
- 括号内容必定有完整的标点体系
- 省略号永远是六个点，从不用两个点...表示停顿
- 缺乏人类书写的"呼吸感"和随意性

### 3. 语言腔调 - "异化表达"
**翻译腔：**
- `"技术上没错，但..."` (Technically correct, but...)
- `"从某种意义上说..."` (In a sense...)
- `"事实上..."` (As a matter of fact...)
- `"值得注意的是..."` (It's worth noting that...)

**书卷气过重：**
- 偏好抽象宏大词汇：`"范式坍缩"`、`"大灭绝"`、`"颠覆性"`
- 回避接地气表达：用`"圣杯"`而非`"标准答案"`
- 喜欢用`"赋能"`、`"重塑"`、`"生态"`等buzzword

**文化错位：**
- 使用非本土化比喻和表达
- 全球化训练背景导致的语境不匹配
- 缺乏地域化的俚语和方言色彩

## 结构层面特征

### 4. 逻辑架构 - "过度工整"
**机械化分点：**
- 强迫症式的`"首先...其次...最后"`
- 过度使用数字编号和分级结构
- 每个段落都有明确的主题句
- 喜欢用`"综上所述"`、`"总的来说"`做总结

**完美平衡：**
- 每个论点都有对应的论据支撑
- 段落长度异常均匀
- 缺乏人类写作的不规整美感
- 转折词使用过于标准化

### 5. 内容组织 - "教科书式"
**说教倾向：**
- 频繁使用`"值得注意的是"`、`"需要强调的是"`
- 过度解释显而易见的概念
- 总想"教育"读者而非平等对话
- 喜欢用`"让我们来看看"`、`"我们可以发现"`

**全知视角：**
- 很少表达不确定性或个人困惑
- 缺乏`"我也不太确定"`、`"可能是我想多了"`等谦逊表达
- 对所有领域都表现出专业性
- 缺乏`"我觉得"`、`"在我看来"`等主观表达

## 情感与个性层面

### 6. 情感表达 - "温和无菌"
**情绪平淡：**
- 缺乏真实的愤怒、兴奋、失望等强烈情感
- 即使表达不同意见也过于礼貌客气
- 缺乏人类的情绪起伏和"毛刺感"
- 很少用脏话或粗俗表达

**安全化表达：**
- 回避争议性表态
- 过度使用缓冲词：`"可能"`、`"或许"`、`"在一定程度上"`
- 对敏感话题绕道而行
- 缺乏强烈的个人立场

### 7. 个性痕迹 - "千人一面"
**缺乏个人标记：**
- 没有独特的口头禅或习惯用词
- 缺乏个人经历和具体细节
- 观点虽多但缺乏独特视角
- 很少提及个人喜好或厌恶

**风格同质化：**
- 不同主题的文章呈现相似的语言风格
- 缺乏专业领域的行话和内部梗
- 写作风格过于"中性"和"平衡"

## 细节层面识别

### 8. 比喻与例证 - "教科书式完美"
**比喻过于工整：**
- 每个比喻都恰好能完美对应要说明的概念
- 缺乏人类比喻的"溢出感"和不完美匹配
- 喜欢用经典比喻：`"冰山一角"`、`"双刃剑"`

**举例模式化：**
- 例子过于经典和安全
- 缺乏新鲜的、个人化的案例
- 经常引用知名企业和历史事件
- 避免使用小众或争议性例子

### 9. 节奏与呼吸 - "机械均匀"
**句长分布：**
- 句子长度过于均匀
- 缺乏短促的感叹或长句的铺陈
- 很少出现单字句或超长句
- 节奏感过于规律

**段落节奏：**
- 段落间转折过于流畅
- 缺乏人类思维的跳跃和突然转向
- 很少出现突然的话题切换
- 逻辑链条过于完整

### 10. 元语言使用 - "过度自省"
**结构化元语言：**
- 频繁使用`"换句话说"`、`"总的来说"`
- 过度标注文本的组织结构
- 喜欢用`"接下来"`、`"现在让我们"`引导

**自指性表达：**
- 经常说明自己在做什么：`"让我们来分析一下"`
- 过度使用框架性语言
- 喜欢预告接下来要说什么

## 综合诊断框架

### 快速识别checklist：
- [ ] 是否频繁使用"A不是X而是Y"句式？
- [ ] 标点使用是否过于"完美"，特别是引号和破折号？
- [ ] 是否有翻译腔和过度书面化？
- [ ] 结构是否过于工整对称？
- [ ] 是否缺乏个人化的表达和细节？
- [ ] 比喻例证是否过于"教科书化"？
- [ ] 整体是否给人"无菌"感？
- [ ] 是否缺乏真实的情感波动？

### 权重评估：
**高权重指标（70%）：**
- 句式模式固化
- 逻辑结构过度完美
- 缺乏真实情感波动

**中权重指标（20%）：**
- 标点使用特征
- 语言腔调异化

**低权重指标（10%）：**
- 细节层面的机械化特征

### 误判风险提醒：
- 某些人类写作（如学术论文、官方文档）可能呈现类似特征
- 经过刻意训练的AI可能减少部分特征
- 需要结合多个维度综合判断，避免单一特征误判
- 不同AI模型可能有不同的"签名特征"
